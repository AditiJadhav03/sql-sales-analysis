&nbsp;Retail Sales SQL Analysis



&nbsp;Project Objective



This project analyzes retail store sales data using SQL to generate business insights.

The objective is to simulate how business analysts and IT support teams monitor store performance, customer behavior, and inventory movement in a retail company.







&nbsp;Dataset



Superstore retail sales dataset containing:



\* Order Date

\* Product Name

\* Category

\* Customer Name

\* Sales Amount

\* Quantity

\* Shipping Mode

\* Region







&nbsp;Tools Used



\* SQLite

\* SQL Queries

\* DB Browser for SQLite







&nbsp;Key Analysis Performed



\* Top selling products

\* Low performing products

\* Monthly sales trends

\* Category-wise revenue

\* Top customers

\* Shipping preference analysis

\* Regional performance







&nbsp;Business Insights



The analysis helps management to:



\* Identify high revenue products

\* Detect slow-moving inventory

\* Understand customer purchasing behavior

\* Optimize logistics operations

\* Monitor monthly and regional performance







&nbsp;Why This Project Matters



Retail companies rely heavily on database queries for reporting, troubleshooting, and operational decision-making.

This project demonstrates practical SQL usage in a real-world retail environment and shows the ability to convert raw data into meaningful business insights.







&nbsp;How to Run



1\. Open DB Browser for SQLite

2\. Open database file `retail.db`

3\. Ensure table `sales` is loaded

4\. Run queries from `queries.sql`
